*   **Context:** High-output solo developer.
*   **Core Logic:** **Data-First Discovery**. We prioritize technical substance over organizational or strategic labels.

## 🛠️ The Discovery Framework (V3.4: Data-First)
We use four interaction levels, prioritizing technical DNA over market tactics:
*   **[L0: State]** - Confirmation of existence (Path/Size).
*   **[L1: Scan]** - **Physical Signature**. (Extensions, Depth, Complexity Metrics).
*   **[L2: Review]** - **Technical DNA**. Identification of imports, class structures, and functional logic.
    *   *Mandate:* Ignore folder labels. Read the code to find the truth.
*   **[L3: Analysis]** - **Strategic Synthesis**. Business Value & ROI identified *only* after technical truth is confirmed.

## 🔍 The Truth Matrix (Source-First)
*   **Truth-Node**: Content matches labels.
*   **Nomad-Node**: Logic identified via **Child-Up Induction** (Trust the imports, not the path).
*   **Ghost-Node**: Intent without substance (Empty/Boilerplate).
*   *Mandate:* Always trust **Discovered-Purpose** over physical path.

## 📊 Measurable Outcomes (KPIs)
1.  **DNA Accuracy**: % of nodes where Agent-inferred technical purpose matches actual code functionality.
2.  **Discovery Precision**: % of Nomad/Drift nodes correctly re-mapped through content analysis.
3.  **Entropy Pruning**: Ratio of functional Source-Truth files vs. filtered artifacts.

## 📑 Universal Output Schema
All discoveries must be reported using this format:
1.  **Discovery Header**: `[LX Discovery] | [Truth-State] | [Engine: XEO/AutoTag]`
2.  **DNA Summary**: Technical breakdown of "what is actually there" (Imports, Logic type).
3.  **Functional Map**: Markdown table showing Path, Technical DNA, and Functional Utility.
4.  **Actionable Details**: Concise (10-20 char) suggestions for next steps.
