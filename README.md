# SupremePower Framework

Welcome to the SupremePower Framework, a consolidated and organized system for managing AI agents, skills, and workflows. This repository houses a collection of powerful tools and conventions designed to enhance productivity and ensure disciplined development.

## Overview

The SupremePower framework integrates components from various sources, including Gemini CLI, Qwen, and custom `.supremepower` configurations, into a unified structure at `/Users/steven/my-supremepowers/`. This consolidation aims to provide a consistent and powerful AI development and operation environment. It now includes dedicated sections for Qwen-specific items (agents, skills, hooks, plugins, debugging, code review, documentation, integrations, plans, projects, security, templates, and various top-level documents) and Gemini-specific items (agents, skills, extensions, policies, configurations, documentation, and foundational documents), alongside consolidated core components and workflows.

The framework is organized into several key areas:

*   **Agents:** Specialized AI components (Gemini CLI, Qwen, custom).
*   **Skills:** Modular, reusable capabilities (Core, Gemini CLI, Qwen).
*   **Rules & Principles:** Governing guidelines for operation and integration.
*   **Scripts:** Utility scripts for automation and management.
*   **Policies:** Security, operational, and behavioral constraints.
*   **Hooks:** Automation logic triggered at specific workflow points (Gemini CLI, Qwen `hookify`, `new-hooks-system`).
*   **Core Framework:** Foundational components enabling orchestration and functionality.
*   **Commands:** Simplified interfaces for skill/agent invocation.
*   **Tests:** Various test suites ensuring framework robustness.
*   **Extensions & Plugins:** Management for Gemini CLI extensions and Qwen plugins.
*   **Workflows:** Documented processes for key operations (feature development, debugging, component integration, request handling, skill internals).
*   **Architectural Comparisons:** Detailed analysis of component origins.
*   **Miscellaneous Top-Level Files:** Important configurations and meta-data from source environments.


The "Suggestions" category of items from `~/.qwen` and `~/.gemini` contained several items that were not found at their source locations and therefore could not be copied. The items that were successfully copied are primarily from the "Need" and "Should" categories. Please refer to the updated Documentation Index for a precise list of all integrated components.
