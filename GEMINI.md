# GEMINI.md - SupremePower Framework Context

This document provides an overview and instructional context for interacting with the SupremePower Framework, designed to bring the power of Gemini CLI directly into the terminal and manage AI agent workflows.

## Project Overview

The `/Users/steven/my-supremepowers/` directory houses a consolidated framework for AI development tools, agents, and comprehensive documentation. It integrates components from Gemini CLI, Qwen, and other sources into a unified system designed for disciplined AI development workflows. This framework is designed for disciplined AI development workflows, emphasizing structured analysis, planning, writing, and systematic debugging.

## Directory Structure and Key Files

The project is organized into several key areas:

*   **`agents/`**: Contains definitions for various AI agents, each with its own documentation. Key files include:
    *   `agent-creation-guidance.md`: Guidance on creating new agents.
    *   `code-reviewer.md`: Instructions for using the code review agent.
    *   `documentation-management.md`: Defines strategies for managing project documentation.
    *   `rule-definition.md`: Guidelines for defining project rules and conventions.
    *   `system-analysis.md`: A framework for conducting in-depth system analysis.
    *   *(Note: Content writing for `documentation-management.md`, `rule-definition.md`, and `system-analysis.md` has encountered persistent tool errors, leaving them empty.)*
*   **`skills/`**: Houses modular, reusable AI skills.
*   **`docs/`**: Contains extensive documentation on various aspects of the framework, including workflows, best practices, and platform integration. Key files include:
    *   `ARCH_COMPARISON.md`: Details the consolidation process and compares intended vs. actual state of the framework.
    *   `README.md`: High-level project overview.
    *   `gemini_cli_supremepower_readme.md`: Specific overview for Gemini CLI integration.
*   **`qwen_` prefixed files/directories**: Indicate components originating from the Qwen ecosystem, covering areas like AI models, workflows, system definitions, and historical integration summaries.
*   **`gemini_` prefixed files**: Indicate components originating from the Gemini CLI ecosystem, covering topics like integration, overview, Git AI, and system configuration.
*   **`go-forward.txt` / `my-powerstxt.txt`**: Logs related to the framework consolidation process.

## Usage Guidelines

This directory acts as a central hub for AI development tools, agents, and documentation. The `GEMINI.md` file itself serves as instructional context for future interactions, guiding the AI agent's operation within this framework. It outlines the expected approach to analysis, planning, writing, and system management, emphasizing discipline, modularity, and clarity. When interacting, prioritize using the specialized agents and skills documented within the `agents/` and `skills/` directories. For any clarification on specific processes or workflows, consult the relevant documentation files in the `docs/` directory or the `GEMINI.md` file itself.
