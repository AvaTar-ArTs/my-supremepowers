# Brainstorming and Design Skill

## Overview
This skill guides the process of transforming initial ideas into well-defined designs and specifications through structured, collaborative dialogue. It emphasizes understanding requirements and validating designs incrementally before implementation.

## Process Flow
1.  **Understand Context:** Analyze the current project state (relevant files, documentation, and recent changes).
2.  **Clarify Requirements:** Ask targeted questions, one at a time, to refine the user's intent, constraints, and success criteria. Prefer multiple-choice questions when feasible.
3.  **Explore Approaches:** Propose 2-3 distinct approaches, outlining their trade-offs and recommending one.
4.  **Present Design:** Break down the design into manageable sections (e.g., 200-300 words per section). Seek user approval after each section. Cover architecture, components, data flow, error handling, and testing.
5.  **Document Design:** Write the validated design into a plan document (e.g., `plans/YYYY-MM-DD-<topic>-design.md`).
6.  **Prepare for Implementation:** Once the design is approved, initiate the next phase by generating a detailed implementation plan (e.g., using a dedicated planning skill).

## Key Principles
*   **Iterative Refinement:** Explore ideas and validate design incrementally.
*   **Focused Inquiry:** Ask one clear question at a time.
*   **Consider Alternatives:** Always propose multiple approaches with trade-offs.
*   **YAGNI:** Ruthlessly remove unnecessary features from designs.
*   **Abstract to General:** Avoid embedding environment-specific commands or agent names; refer to abstract actions (e.g., "use a planning skill").

## When to Use
Use this skill when embarking on any creative work, including developing new features, building components, adding functionality, or modifying behavior. It is essential for exploring user intent, requirements, and design before any implementation begins.
